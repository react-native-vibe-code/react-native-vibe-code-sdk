# Features

React Native Vibe Code is a comprehensive AI-powered development environment for building React Native and Expo mobile applications through natural language. This guide covers all major features with practical examples.

## Core Features

### 1. AI-Powered Code Generation

Generate complete mobile applications using natural language with Claude Code SDK integration.

**Models Available:**
- **Claude Opus 4.5** (Default) - Most powerful for complex apps
- **Claude Sonnet 4.5** - Balanced performance and speed
- **Claude Haiku 4.5** - Fast for simple apps

**Example:**
```typescript
// User input: "Create a todo app with dark mode"

// AI generates:
// - Complete React Native/Expo project structure
// - Todo components with CRUD operations
// - Dark mode theme with AsyncStorage persistence
// - Type-safe code with TypeScript
// - Proper navigation setup
```

**Key Capabilities:**
- Full app scaffolding
- Component generation
- State management setup
- API integration
- Navigation implementation
- Styling with Tailwind/NativeWind

**Documentation:** [AI SDK Integration](/ai-sdk-integration)

### 2. E2B Sandbox Execution

Secure, isolated code execution environments for building and testing apps.

**Features:**
- **Isolated Environments** - Each project runs in its own sandbox
- **Real-time File System** - Watch files change as AI codes
- **Package Management** - Automatic npm/pnpm/yarn installation
- **Multiple Templates** - Expo, Tamagui, GitHub integration
- **Hot Reload** - See changes instantly in preview

**Sandbox Lifecycle:**
```typescript
// 1. Create project
const project = await createProject(userId, 'My Todo App')

// 2. Sandbox auto-created with template
// - Expo template with React Native
// - TypeScript configured
// - Dependencies pre-installed
// - File watcher active

// 3. AI writes code to sandbox
await sandbox.files.write('app/todo.tsx', todoComponent)

// 4. Preview updates in real-time
// http://[sandbox-id].e2b.dev
```

**Available Templates:**
- `expo-template` - Standard Expo app with TypeScript
- `tamagui-template` - Expo + Tamagui UI library
- `github-template` - GitHub integration support

**Template Location:** `sandbox-templates/`

### 3. Voice Input

Speak your app ideas instead of typing with OpenAI Whisper transcription.

**Features:**
- **Real-time Recording** - Browser-based audio capture
- **Multi-language Support** - Whisper supports 50+ languages
- **Automatic Transcription** - Convert speech to text instantly
- **Noise Cancellation** - Echo and noise suppression
- **Mobile Optimized** - Works on iOS and Android browsers

**React Hook:**
```typescript
import { useAudioRecorder } from '@/hooks/use-audio-recorder'

function VoiceInput() {
  const {
    isRecording,
    startRecording,
    stopRecording,
  } = useAudioRecorder()

  const handleVoiceInput = async () => {
    if (isRecording) {
      const audioBlob = await stopRecording()
      const formData = new FormData()
      formData.append('audio', audioBlob)

      const response = await fetch('/api/transcribe', {
        method: 'POST',
        body: formData
      })

      const { text } = await response.json()
      // Use transcribed text
    } else {
      await startRecording()
    }
  }

  return (
    <button onClick={handleVoiceInput}>
      {isRecording ? 'Stop' : 'Speak'}
    </button>
  )
}
```

**Audio Settings:**
```typescript
{
  echoCancellation: true,
  noiseSuppression: true,
  sampleRate: 16000,      // Optimized for speech
  channelCount: 1,        // Mono audio
}
```

**Max File Size:** 25MB
**Max Duration:** 2 minutes
**Supported Formats:** WebM, MP4, WAV, MPEG

**Documentation:** [Voice Chat](/voice-chat)

### 4. Skills System

Pre-built API integrations that can be added to any project with a single click.

**Available Skills:**

#### AI Chat (Claude)
Add conversational AI to your app using Claude models.

**Skill ID:** `anthropic-chat`

**Generated Code:**
```typescript
import { generateText } from 'ai'
import { createAnthropic } from '@ai-sdk/anthropic'

const anthropic = createAnthropic({
  apiKey: process.env.ANTHROPIC_API_KEY
})

async function chatWithClaude(messages) {
  const { text } = await generateText({
    model: anthropic('claude-3-5-haiku-20241022'),
    messages: messages
  })
  return text
}
```

### 5. Real-time File Watching

Monitor file changes in the sandbox with Pusher integration.

**Features:**
- **Live Updates** - See files change as AI codes
- **File Tree Sync** - Automatically update project structure
- **Change Notifications** - Get notified on important changes
- **Collaborative Editing** - Multiple users can watch same project

**Implementation:**
```typescript
import Pusher from 'pusher-js'

const pusher = new Pusher(process.env.NEXT_PUBLIC_PUSHER_KEY, {
  cluster: process.env.NEXT_PUBLIC_PUSHER_CLUSTER
})

const channel = pusher.subscribe(`project-${projectId}`)

channel.bind('file-changed', (data) => {
  console.log('File updated:', data.path)
  // Refresh file in UI
})

channel.bind('file-created', (data) => {
  console.log('New file:', data.path)
  // Add to file tree
})

channel.bind('file-deleted', (data) => {
  console.log('File removed:', data.path)
  // Remove from file tree
})
```

**File:** `lib/sandbox-file-watcher.ts`

### 7. Project Management

Full CRUD operations for projects with database persistence.

**Create Project:**
```typescript
const project = await db.insert(projects).values({
  id: generateId(),
  userId: user.id,
  name: 'My Todo App',
  description: 'A simple todo app',
  template: 'expo-template',
  status: 'active',
})
```

**List Projects:**
```typescript
const userProjects = await db
  .select()
  .from(projects)
  .where(eq(projects.userId, userId))
  .orderBy(desc(projects.createdAt))
```

**Update Project:**
```typescript
await db
  .update(projects)
  .set({
    name: 'Updated Name',
    visibility: 'public',
  })
  .where(eq(projects.id, projectId))
```

**Delete Project:**
```typescript
await db
  .update(projects)
  .set({ status: 'deleted' })
  .where(eq(projects.id, projectId))
```

**Database Schema:** `lib/db/schema.ts`

```typescript
export const auth = betterAuth({
```

**Client Usage:**
```typescript
import { useSession } from '@/lib/auth/client'

function ProfileButton() {

  return (
    <div>
      <img src={session.user.image} />
      <span>{session.user.name}</span>
```

### 15. Rate Limiting

**Features:**
- **Per-user Limits** - Based on subscription tier
- **Message Counting** - Track API usage
- **Graceful Degradation** - Show upgrade prompts
- **Usage Dashboard** - View current usage

**Implementation:**
```typescript
import { canUserSendMessage, incrementMessageUsage } from '@/lib/message-usage'

const usageCheck = await canUserSendMessage(userId)

if (!usageCheck.canSend) {
  return {
    error: 'Rate limit exceeded',
    usage: usageCheck.usage,
    upgradeUrl: '/pricing'
  }
}

// Process message...

await incrementMessageUsage(userId)
```
